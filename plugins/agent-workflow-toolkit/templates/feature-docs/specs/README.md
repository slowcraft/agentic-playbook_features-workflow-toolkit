# Specs — <feature>

Canonical artifacts that describe `<feature>`'s shape after decisions have been settled.

- **Brainstorms** (typically in chat history or developer notes) capture exploration
- **Specs** capture the resulting artifacts: "this is what we built / are building"

When a brainstorm settles into a decision, the spec is the resulting artifact — not an edit to the brainstorm.

## Typical specs for a feature

| Spec | Purpose |
|---|---|
| `<output>_contract.md` | The canonical wire format / data model your feature produces or consumes |
| `test_scenarios_catalog.md` | If your feature has an eval target: rows × criteria × verdicts with stable IDs |
| `<feature>_implementation_plan.md` | Phase split for the biggest implementation story |
